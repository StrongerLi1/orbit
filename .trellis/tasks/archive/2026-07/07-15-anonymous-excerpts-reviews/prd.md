# 为摘录和书评增加匿名功能

## Goal

允许用户在 Web 端发布摘录或书评时选择匿名展示，降低公开分享阅读感受时的身份暴露，同时保留现有的归属校验、编辑/删除权限和共享内容能力。

## Background and scope

- 摘录在 `excerpts` 表中保存创建者 UID 和用户名快照；所有已登录用户可以读取，服务端按归属用户或管理员权限控制编辑、删除。
- 书评在 `book_reviews` 表中保存创建者 UID 和用户名快照；所有有 `library:read` 权限的用户可以读取，服务端按归属用户或管理员权限控制删除。
- 浏览器端已有摘录列表、摘录新增/编辑表单和书评发布/删除弹窗。
- 现有数据库升级和 JSON 导入路径需要保持兼容；历史记录当前没有匿名状态，按非匿名处理。
- 本次只覆盖 Web 端摘录和书评；Android 摘录和书评不在范围内，后续单独处理。

## Requirements

- 摘录和书评均支持独立记录的匿名开关，匿名状态由服务端持久化。
- 摘录的匿名状态在现有编辑表单中可切换；书评正文仍不支持编辑，但作者可通过独立操作切换匿名状态。
- 摘录的匿名状态沿用现有摘录编辑权限；书评匿名切换只允许书评作者本人，服务端按真实 UID 校验，不能依赖前端隐藏按钮。
- 创建请求和更新请求不接受客户端传入或修改作者 UID/用户名；作者身份始终来自认证上下文。
- 对其他普通读取者，匿名摘录/书评不显示真实用户名，统一显示“匿名用户”；作者本人看到真实用户名及“匿名”标识；管理员可看到真实用户名，并保留现有管理能力。
- 非匿名记录继续显示现有用户名快照；匿名不改变内容、出处、时间、记录归属或现有编辑/删除权限。
- 书评匿名切换成功后刷新书评列表；摘录匿名状态随编辑保存并刷新摘录列表。

## Acceptance Criteria

- [x] 创建匿名摘录后，其他普通读取者看到“匿名用户”且看不到真实用户名；创建者看到真实用户名和“匿名”标识，且仍可编辑/删除。
- [x] 创建匿名书评后，其他普通读取者看到“匿名用户”且看不到真实用户名；作者看到真实用户名和“匿名”标识，且仍可删除。
- [x] 摘录编辑和书评匿名切换都能在匿名与非匿名之间往返；关闭匿名后读取恢复显示用户名，正文和归属不变。
- [x] 普通用户不能通过请求体伪造作者、修改他人摘录的匿名状态、修改他人书评匿名状态或借匿名状态绕过删除权限；管理员保留现有摘录管理和书评删除能力。
- [x] 旧摘录、旧书评和旧数据库启动升级正常，历史记录按非匿名兼容处理，JSON 导入记录也能正常写入匿名默认值。
- [x] Web 端摘录表单、摘录卡片、书评弹窗和书评操作正确反映匿名状态；后端测试、浏览器语法检查和项目测试通过。

## Out of scope

- Android 摘录匿名功能。
- Android 共享图书馆和书评功能。
- 书评正文编辑、匿名评论审核、匿名统计或匿名用户资料。
