# 前端视觉重设计方案

## 1. 设计意图

Orbit 不是营销站，也不是通用 AI 控制台。它是一个会频繁打开的个人工作台。视觉设计优先让用户快速看清今天要做什么、保存了什么、系统当前处于什么状态，同时保留一点书房的温度。

设计关键词：现代书房、暖灰、橄榄、亚麻、柔和色块、长期使用。

避免关键词：编辑排版样张、糖果色效率面板、密集细线、未来感、玻璃、发光、同质卡片墙、装饰性英文。

## 2. 改动边界与稳定契约

### HTML

- 可以重排 `public/index.html` 的展示结构、分组和文案。
- 必须保留 `public/app.js` 使用的元素 `id`、`data-*` 属性和可见性入口。
- 导航图标可替换为统一的轻量内联 SVG 或简洁文字标记，不引入图标依赖。
- 纯装饰模块可以删除或改成有信息价值的内容；不能移除功能入口。

### CSS

- 重写 `public/styles.css`，按 token、基础元素、框架、通用组件、页面模块、响应式顺序组织。
- 保留 JS 使用的状态类：`active`、`open`、`show`、`done`、`disabled`、`managing`、`dragging`、`drag-over` 及业务状态类。
- PlayCaptcha 只通过现有 CSS 变量和外围容器适配新视觉，不修改 island 行为。

### JavaScript

- `public/app.js` 只允许修改展示字符串、图标字符和必要的视图类名。
- 不改变请求地址、请求顺序、事件语义、权限判断、状态模型和流式聊天逻辑。

## 3. 视觉系统

### 色彩

- 页面底色：暖灰米色，不使用渐变。
- 内容底色：象牙白；关键首页模块使用鼠尾草、亚麻和灰绿色区分信息角色。
- 主文字：深橄榄灰；次要文字使用中性暖灰，保持对比度。
- 主操作色：亚麻色用于主要按钮和当前导航，深亚麻棕用于链接、焦点和小型状态标记。
- 辅助色：鼠尾草用于问候与完成状态，亚麻色用于随机摘录。
- 边界色只用于表单、表格和必要分隔，主要层级由背景色与阴影承担。

### 字体

- UI 与正文使用系统无衬线中文字体栈；页面标题、问候和摘录使用系统宋体/衬线字体栈。
- 衬线字体只用于需要阅读感的位置，导航、按钮、表单和数据仍保持清楚的无衬线字体。
- 不再使用全大写英文眉题表达层级；层级由字号、字重、行距和位置建立。

### 形状与质感

- 基础圆角控制在 7–17px；标签和状态徽记仅在语义需要时使用胶囊形。
- 常规内容使用柔和背景、留白和轻阴影分组；细线退到列表内部和表单边界。
- 动效只用于页面进入、侧栏开合、浮层和控件反馈，时长短且不影响阅读。

## 4. 页面框架

### 桌面端

- 左侧导航使用深橄榄灰底色，当前入口使用亚麻色块，和暖灰内容区形成稳定对比。
- 品牌、主导航和管理导航形成明确分组；底部删除励志引语，改为简短的产品范围说明和当前用户信息。
- 顶栏缩小高度，搜索框融入工具栏；日期、用户和退出操作保持可见但降低视觉噪音。
- 内容区使用稳定的最大宽度和更清楚的纵向节奏。

### 移动端

- 延续现有 `mobile-menu` 与 `.sidebar.open` 行为，将侧栏作为带遮罩的抽屉呈现。
- 顶栏保留菜单、搜索和用户入口；次要信息可隐藏或换行。
- 页面标题操作区堆叠，主要操作保持易点按；复杂表格/统计改为纵向信息行。

## 5. 通用组件

- 按钮：主按钮为实色矩形，次按钮为描边或文字按钮；统一高度、焦点环和禁用状态。
- 表单：使用可见标签、清楚边框和纯色焦点状态；不使用发光阴影。
- 数据容器：`card` 不再默认承担所有页面分组，只为聊天区、独立统计和需要背景隔离的区域提供边界。
- 标签/筛选：分类筛选保留胶囊语义，其余状态尽量用文字、色点和细线表达。
- 弹窗：使用清晰标题、操作区分隔和有限阴影；验证码外观通过变量协调。
- 空状态：说明当前缺少什么，并给出可执行的下一步；不使用鼓励式口号。
- Toast：作为短暂操作结果，保持高对比但不使用漂浮发光效果。

## 6. 页面级编排

### 登录/注册

- 改成左右不对称的双色版式：深橄榄品牌说明区 + 象牙白表单区；手机端合并为单列。
- 文案直接说明 Orbit 保存的内容和登录目的，删除 `PRIVATE ORBIT` 等英文装饰。

### 概览

- 删除装饰性轨道星球和泛化励志块。
- 首屏并列展示鼠尾草色个性化问候和亚麻色随机摘录，并提供“换一条”操作。
- 日期、快速添加、低饱和统计、今日计划、待办和最近收藏形成清楚的阅读顺序。
- 统计使用三种相近的自然软色；计划、待办和收藏使用不同的信息密度，不全部做成同款卡片。

### 收藏、摘录、计划、待办与网盘

- 收藏使用更像目录的网格/列表混合结构，域名、备注、分类和操作层级清晰。
- 摘录保留衬线阅读感，但减少装饰性大引号和卡片瀑布的模板感。
- 计划将日期导航、当天执行、七日统计和按计划统计组织为同一工作区。
- 待办以清楚的进行中/已完成分区呈现，完成项降低视觉权重但仍可操作。
- 网盘搜索突出搜索、来源筛选、结果与安全提示，不把每条结果做成漂浮卡片。

### Hermes 与管理页面

- Hermes 聊天保持会话列表、消息区和输入区三段结构，消息气泡减少圆润聊天应用模板感。
- Hermes 状态、用户管理和聊天审计使用偏工具化的信息行与状态标记，提高数据密度。
- 危险操作继续使用明确的红色语义，不因视觉统一而弱化。

## 7. 文案策略

- 删除所有仅作装饰的英文眉题和大写标签。
- 把泛化句子改为任务导向说明，例如“慢一点也没关系”改为具体的日期/任务摘要。
- 保留产品名、Hermes、PanSou、角色名、权限名和技术地址等专有词。
- 空状态采用“当前状态 + 下一步”结构；错误与风险提示保持直接。
- 完成初稿后按 humanizer 规则做一次反模板审校，删除营销语气、机械排比和无信息量结尾。

## 8. 兼容性与回滚

- CSS 采用原生布局能力，不新增构建步骤；支持当前现代浏览器。
- 缓存版本参数在视觉文件更新后同步修改，避免浏览器继续加载旧样式。
- HTML、CSS、JS 分文件提交范围清楚，可通过 Git 按文件回退；业务代码无迁移步骤。
- 若新布局暴露交互依赖问题，优先恢复对应 DOM 契约，不修改业务逻辑绕过。

## 9. 验证策略

- 静态检查：检查 JS 选择器依赖、重复 `id`、缺失 `data-*`、语法和测试。
- 视觉检查：1440×1000 与 390×844 下检查登录、概览、至少一个内容页、Hermes 聊天、管理页和弹窗。
- 状态检查：空、有数据、hover、focus-visible、disabled、错误、成功、侧栏展开、弹窗和长文本。
- 若本地数据库不可用，使用不提交的临时静态数据预览关键已登录页面；不得把临时预览代码带入产品文件。
