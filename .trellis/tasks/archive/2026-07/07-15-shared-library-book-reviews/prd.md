# 共享图书馆书评功能

## Goal

让共享图书馆的用户可以围绕书籍留下可见的书评，并在提交阅读记录时方便地补充书评；评论权限由服务端按评论归属和管理员身份控制。

## Confirmed facts

- 共享图书馆书籍存储在 `books` 表，阅读记录存储在 `book_reads` 表。
- `book_reads` 当前包含书籍、用户、阅读日期和时间戳，没有书评字段。
- 后端已有 `/api/library/books/{book_id}/reads` 的阅读记录查询、创建、更新和删除接口。
- 浏览器端已有阅读日期表单和按用户展示阅读记录的弹窗；提交阅读记录时目前只发送 `readDate`。
- 浏览器端已经通过 `library:read` 权限保护共享图书馆访问，阅读记录只能由记录所属用户修改或删除。
- 现有管理员身份和权限体系可用于管理员删除全部书评。
- Android 客户端当前没有共享图书馆页面或对应 API 封装，本任务只覆盖当前已有的浏览器端共享图书馆。

## Requirements

- 所有有权访问共享图书馆的用户，无论当前是否已读某本书，都可以提交该书的书评。
- 书评需要记录所属用户，并在书籍的书评区域展示评论内容和评论用户名。
- 普通用户只能删除自己的书评；管理员可以删除该书的全部书评。
- 删除权限必须由后端校验，不能只依赖前端隐藏按钮。
- 用户提交阅读记录时提供可选书评字段；不填写书评也能成功提交阅读记录。
- 书评作为独立记录归属于书籍和用户；从阅读记录表单填写书评时，同时创建一条独立书评，不与某条阅读记录绑定。
- 编辑阅读日期时不修改已有书评；只有在创建阅读记录时填写的书评会创建新评论。
- 书评只支持发布和删除，不支持编辑；需要修改时由用户删除后重新发布。
- 书评和阅读记录的 API、数据库迁移和浏览器 UI 保持现有认证、错误处理和刷新模式。

## Acceptance Criteria

- [x] 用户未读某书时仍可提交书评，用户已读某书时也可提交书评。
- [x] 书评列表显示评论用户名、内容和创建时间，并能区分当前用户自己的评论。
- [x] 普通用户删除自己的评论成功；删除他人评论返回拒绝结果且数据不变。
- [x] 书评不提供编辑入口；用户可以删除自己的评论后重新发布。
- [x] 管理员可以删除任意用户的评论。
- [x] 新增阅读记录时，书评字段可留空；填写后 API 和书评列表均能看到该内容。
- [x] 现有仅包含阅读日期的阅读记录继续正常读取和编辑，不因新增书评字段失败；删除阅读记录不会删除独立书评。
- [x] 页面在创建或删除评论/阅读记录后刷新相关书籍的阅读统计和书评列表。
- [x] 后端和浏览器端通过对应语法、单元和集成检查。

## Out of scope

- 书评评分、点赞、回复、编辑和审核流程。
- 面向未登录用户开放书评。
- 将共享图书馆改为按用户隔离的私有书架。
- Android 端共享图书馆和书评功能，留给后续 Android 图书馆功能任务。

## Notes

这是跨数据库、FastAPI 和浏览器 UI 的复杂任务；`design.md` 和 `implement.md` 已补充，进入实现前需要用户审核规划产物。
